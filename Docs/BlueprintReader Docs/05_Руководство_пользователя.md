# Руководство пользователя — BlueprintReader

## Установка и активация плагина

BlueprintReader распространяется через **Fab** (маркетплейс Epic Games) как редакторный плагин для Unreal Engine. Продукт полностью бесплатен для личного использования и открыт по лицензии **MIT** — исходный код доступен на GitHub, вы можете его читать, модифицировать и встраивать в свои инструменты. Для коммерческого/студийного использования на Fab установлена символическая цена **$10** — эта модель "бесплатно для личных проектов, недорого для бизнеса" осознанно выбрана авторами и является одним из ключевых отличий плагина от типичных платных решений на маркетплейсе.

Чтобы установить плагин:

1. Откройте страницу BlueprintReader на Fab и нажмите **Add to Library** (или сразу **Add to Project**, если у вас уже открыт нужный проект в Epic Games Launcher).
2. В Epic Games Launcher выберите проект, в который нужно добавить плагин, и подтвердите установку. Launcher скопирует плагин в папку `Plugins` вашего проекта (либо установит его как engine-плагин, если вы устанавливали его глобально для движка).
3. Если вы получили плагин не через Launcher, а, например, скачали архив с GitHub — распакуйте его в подпапку `Plugins/BlueprintReader` внутри папки вашего проекта (там же, где лежит `.uproject` файл). Структура должна выглядеть так: `YourProject/Plugins/BlueprintReader/BlueprintReader.uplugin`.
4. Запустите (или перезапустите) Unreal Editor для вашего проекта. При первом запуске после установки движок предложит пересобрать модули плагина — согласитесь, это займёт от нескольких секунд до пары минут в зависимости от вашей машины.
5. Убедитесь, что плагин включён: откройте **Edit → Plugins**, найдите категорию **Other**, а в ней — **BlueprintReader**. Флажок **Enabled** должен быть установлен. Плагин помечен как `EnabledByDefault`, поэтому в норме включать его вручную не требуется — но если вы после установки не видите нового пункта в контекстном меню, стоит проверить именно этот флажок.

**Поддерживаемые версии движка.** Плагин собирается из единого исходного дерева и поддерживает **Unreal Engine 5.7 и 5.8** без каких-либо действий с вашей стороны — просто установите версию, соответствующую вашему движку (на Fab это будут отдельные "Project Version" под каждую версию движка). Если вы используете более старую версию UE (5.6 и ниже), плагин может не собраться — авторы не гарантируют обратную совместимость с более ранними релизами.

**Платформы.** Плагин официально поддерживается на Windows, macOS и Linux, поскольку это чисто редакторный C++/Slate инструмент, не зависящий от платформы таргета вашей игры.

После включения плагина никакой дополнительной настройки не требуется — можно сразу переходить к разделу «Быстрый старт».

## Быстрый старт

Идея BlueprintReader предельно проста: вы кликаете правой кнопкой мыши на ассете в Content Browser — и получаете аккуратный, читаемый Markdown-текст с описанием его структуры, логики и (для виджетов) визуальной компоновки. Этот текст можно скопировать и вставить в диалог с любым AI-ассистентом (ChatGPT, Claude, Gemini и т.д.), чтобы получить помощь в отладке, рефакторинге, документировании или просто в объяснении «что делает этот Blueprint».

Шаги для первого запуска:

1. Откройте **Content Browser** в редакторе.
2. Найдите любой Blueprint-ассет, который вы хотите изучить — например, Actor Blueprint персонажа или Widget Blueprint интерфейса.
3. Кликните по нему **правой кнопкой мыши**, чтобы открыть контекстное меню.
4. В открывшемся меню найдите пункт **"Read Blueprint for AI-Assistant"** (в секции действий с ассетом) и выберите его.
5. Если тип ассета поддерживается — откроется отдельное окно плагина с вкладками (например, **Structure** и **Graph**, а для виджетов ещё и **Design**). Если ассет пока не поддерживается — вместо этого появится небольшое информационное окно с пояснением и ссылкой на GitHub-репозиторий проекта, где можно проверить обновления.
6. Внутри окна переключайтесь между вкладками, чтобы посмотреть разные срезы информации об ассете: общая структура (переменные, компоненты, функции), логика графов (события, ноды, связи) и, для UMG-виджетов, визуальная иерархия элементов интерфейса.
7. Когда нужный текст найден — либо выделите и скопируйте его прямо из окна, либо воспользуйтесь кнопкой **"Export to file…"** внизу окна, чтобы сохранить весь результат в файл `.md` или `.txt` (подробнее — в разделе «Экспорт результатов»).
8. Вставьте полученный текст в чат с вашим AI-ассистентом вместе с вопросом — например, «объясни, что делает этот Blueprint» или «помоги найти причину бага в этой логике».

Важный момент: BlueprintReader **не отправляет ничего никуда сам**. Он не подключается ни к каким LLM API и не требует ключей доступа — это осознанное архитектурное решение авторов. Плагин лишь готовит чистый, компактный текст; выбор AI-инструмента, в который вы этот текст вставите, полностью остаётся за вами.

## Поддерживаемые типы ассетов

На данный момент BlueprintReader умеет "читать" следующие типы ассетов:

- **Actor Blueprints** — любые Blueprint-классы, унаследованные от `Actor` (персонажи, интерактивные объекты, пикапы и т.д.).
- **ActorComponent Blueprints** — Blueprint-классы компонентов, которые вы добавляете к акторам.
- **Widget Blueprints (UMG)** — экраны и элементы интерфейса, включая как "листовые" виджеты, так и переиспользуемые пользовательские виджеты (`WBP_*`), вложенные друг в друга.
- **Blueprint Interfaces (BPI)** — интерфейсы Blueprint, описывающие набор функций-контрактов без реализации.
- **User-Defined Enums** — перечисления, созданные через редактор (не C++ enum).
- **User-Defined Structures** — пользовательские структуры данных, созданные через редактор.
- **Materials** — обычные материалы и инстансы материалов.
- **Material Functions** — материальные функции и их инстансы, то есть переиспользуемые куски логики материалов.

Для каждого из этих типов плагин показывает разный набор вкладок, потому что смысл информации отличается:

- Для **Widget Blueprint** доступны три вкладки: **Design** (визуальная компоновка экрана — какие элементы где расположены, их размеры и ключевые визуальные свойства), **Structure** (переменные, привязки событий) и **Graph** (логика на графах событий и функций). Вкладка Design открывается первой, потому что для интерфейсов чаще всего в первую очередь важно именно "что где нарисовано".
- Для **Actor**, **ActorComponent**, **Material**, **Material Function** и **Blueprint Interface** доступны две вкладки: **Structure** и **Graph**.
- Для **Enum** и **Structure** — только одна вкладка **Structure**, поскольку у этих типов ассетов просто нет логики графов или визуальной компоновки.

**Что плагин пока не поддерживает.** Если вы попробуете открыть, например, обычный не-акторный и не-компонентный Blueprint без явного подтипа, Anim Blueprint, звуковой ассет, текстуру, Data Asset, Data Table, Niagara-систему или ассет уровня — плагин честно сообщит, что этот тип пока не реализован, вместо того чтобы показать пустое или некорректное окно. Это сделано специально: лучше явно сказать "ещё не поддерживается", чем выдать вводящий в заблуждение результат. Список поддерживаемых типов будет расширяться в будущих версиях — следить за обновлениями можно на странице проекта на GitHub, ссылка на которую есть прямо во всплывающем окне для неподдерживаемых ассетов.

## Настройки плагина

Настройки BlueprintReader находятся в стандартном месте для плагинов Unreal Engine: **Edit → Project Settings → Plugins → Blueprint Reader**. Там доступны три параметра:

- **Export Format** — формат, в котором будет сохраняться файл при экспорте результатов: **Markdown (.md)** или **Plain Text (.txt)**. По умолчанию выбран Markdown — он лучше подходит для вставки в чаты с AI-ассистентами, поскольку большинство из них корректно распознают Markdown-разметку (заголовки, списки, блоки кода) и отображают её читаемо. Plain Text полезен, если вам нужен максимально "сырой" текст без разметки — например, для вставки в системы, которые не поддерживают Markdown.
- **Restrict Widget Recursion Depth** — включает или отключает ограничение на глубину обхода вложенных виджетов. По умолчанию включено.
- **Widget Recursion Depth** — собственно значение этой глубины, от 1 до 50 (по умолчанию 10). Этот параметр становится доступен для редактирования только тогда, когда включена галочка выше.

Зачем нужна эта настройка глубины? UMG-виджеты часто вкладываются друг в друга: экран содержит переиспользуемый виджет карточки персонажа, та — виджет полоски здоровья, та, в свою очередь, — ещё какие-то элементы, и так далее. Если позволить плагину раскрывать эту вложенность бесконечно, результат для сложного проекта может превратиться в огромную "простыню" текста, которую неудобно читать и накладно вставлять в диалог с AI-ассистентом (у большинства чат-моделей есть практический лимит на объём контекста за одно сообщение). Ограничение глубины — это разумный компромисс: плагин раскрывает вложенность виджетов до заданного уровня, а глубже — просто перестаёт углубляться, оставляя структуру компактной и читаемой.

**Как выбрать значение.** Если вы работаете с простым UI (пара уровней вложенности) — значение по умолчанию (10) с огромным запасом покрывает любой реальный экран, и вы, скорее всего, никогда не столкнётесь с ограничением на практике. Если же вы разрабатываете что-то с очень глубокой вложенностью переиспользуемых виджетов и замечаете, что вывод обрывается раньше, чем ожидалось — увеличьте значение либо полностью снимите галочку **Restrict Widget Recursion Depth**, чтобы снять ограничение вовсе. Обратный сценарий тоже возможен: если вас интересует только верхнеуровневая компоновка экрана без глубоких деталей вложенных виджетов, можно, наоборот, уменьшить глубину — это сделает вывод короче и сфокусированнее.

Изменения в настройках применяются сразу к следующему запуску извлечения данных — не нужно перезапускать редактор.

## Экспорт результатов

Помимо чтения результата прямо в окне плагина, вы можете сохранить его в файл — это удобно, если вы хотите:

- приложить файл к тикету в баг-трекере или к сообщению коллеге;
- сохранить "снимок" состояния Blueprint на определённый момент, чтобы позже сравнить с новой версией;
- вставить содержимое файла в AI-инструмент, который принимает файлы, а не только текст в чате (например, при работе с ассистентами, встроенными в IDE).

Для этого в нижней части окна плагина есть кнопка **"Export to file…"**. Она видна на любой вкладке — панель с кнопкой не пропадает при переключении между Structure/Graph/Design, потому что живёт в общем "подвале" окна.

При нажатии на кнопку происходит следующее:

1. Плагин собирает воедино все содержательные разделы результата (Structure, Graph, Design — для тех вкладок, которые применимы к текущему типу ассета), пропуская пустые или неактуальные для данного типа секции (например, для материала не будет раздела Design, и он просто не попадёт в файл).
2. Если сохранять реально нечего — вы увидите уведомление **"Nothing to export for this asset."**, и диалог сохранения не откроется.
3. Если есть, что сохранять — откроется стандартное системное окно **"Export Blueprint Data"** для выбора места и имени файла. Расширение файла (`.md` или `.txt`) и фильтр типов файлов в этом диалоге автоматически подставляются в соответствии с параметром **Export Format** из настроек плагина (см. предыдущий раздел).
4. По умолчанию предлагаемое имя файла берётся из имени самого ассета (например, `BP_MyCharacter.md`), так что в большинстве случаев можно просто нажать «Сохранить», ничего не переименовывая.
5. Если вы передумали и закрыли диалог сохранения без выбора файла — ничего не произойдёт, никакого уведомления об ошибке не будет: это штатное поведение, а не сбой.
6. После успешного сохранения появится уведомление об успехе с именем файла. Если что-то пошло не так (например, недостаточно прав на запись в выбранную папку) — появится уведомление об ошибке экспорта.

Файл сохраняется в кодировке UTF‑8, поэтому кириллица, спецсимволы и любые нестандартные имена переменных из вашего проекта отображаются корректно в любом текстовом редакторе.

## Частые вопросы (FAQ)

**Плагин действительно бесплатный?**
Да. BlueprintReader — открытый проект с лицензией MIT, доступный на GitHub. На Fab он бесплатен для личного/некоммерческого использования. Если вы используете его в рамках студии/бизнеса, авторы просят приобрести лицензию за символическую сумму $10 — это скорее способ поддержать разработку, чем полноценная коммерческая модель.

**Нужен ли мне API-ключ OpenAI/Anthropic/Google, чтобы это работало?**
Нет. BlueprintReader не обращается ни к каким внешним LLM-сервисам и не требует никаких ключей доступа. Он только готовит текстовое описание вашего ассета — вставлять этот текст в конкретного AI-ассистента (в браузере, в приложении, в IDE) вы делаете сами, вручную, тем инструментом, каким пользуетесь. Это принципиальное архитектурное решение: плагин не "привязывает" вас к одному провайдеру AI и не собирает никаких данных о том, куда вы вставили результат.

**Почему при клике на некоторые ассеты появляется не окно с результатом, а маленькое информационное окошко?**
Это значит, что тип ассета пока не входит в список поддерживаемых (см. раздел «Поддерживаемые типы ассетов»). Информационное окно так и говорит: это не ошибка движка, а просто ещё не реализованная функциональность. В окне есть кнопка "Check for updates" — она ведёт на страницу проекта на GitHub, где можно проверить, не появилась ли уже поддержка нужного типа в новой версии.

**Планируется ли поддержка Anim Blueprint, Niagara, Data Table и других типов?**
Список поддерживаемых типов активно расширяется. Актуальный статус и планы всегда можно посмотреть в репозитории проекта на GitHub — именно там ведётся открытая разработка, и любые новые релизы сначала появляются там же, откуда синхронизируются на Fab.

**Чем это отличается от похожих плагинов на Fab (например, от инструментов класса "Blueprint → Markdown/AI")?**
Главное отличие — открытость и модель распространения: BlueprintReader полностью open-source (MIT), бесплатен для личного использования, и не привязывает вас к какому-либо конкретному AI-провайдеру или встроенному API-ключу. Дополнительно у плагина есть отдельная вкладка **Design**, которая специально показывает визуальную компоновку UMG-виджетов — не только логику графов, но и то, как расположены элементы интерфейса, что особенно полезно при обсуждении UI/UX с AI-ассистентом.

**Можно ли использовать результат экспорта как официальную документацию проекта?**
Формально — да, файл получается читаемым и структурированным, и ничто не мешает приложить его к вики проекта. Но стоит понимать: основная цель плагина — подготовка контекста для AI-ассистента, а не замена полноценной ручной документации. Для критичных, долгоживущих частей проекта мы рекомендуем всё же поддерживать документацию, написанную и проверенную человеком.

**Могу ли я редактировать Blueprint через этот плагин?**
Нет. BlueprintReader — это инструмент только для чтения (read-only). Он не изменяет ваши ассеты никаким образом, а лишь извлекает и показывает информацию о них. Любые изменения в Blueprint вы, как и раньше, вносите через стандартный редактор Unreal Engine.

## Устранение неполадок

**В контекстном меню Content Browser нет пункта "Read Blueprint for AI-Assistant".**
Проверьте, что плагин включён: **Edit → Plugins → Other → BlueprintReader**, флажок **Enabled**. Если флажок пришлось ставить вручную — перезапустите редактор, чтобы модуль плагина корректно подключился к системе меню Content Browser. Также убедитесь, что вы кликаете правой кнопкой именно по ассету (а не, например, по пустому месту панели или по папке) — пункт меню добавляется в контекст действий с конкретным ассетом.

**После клика открывается не окно с результатом, а информационное окошко про неподдерживаемый тип.**
Это ожидаемое поведение для ассетов, тип которых пока не входит в список поддерживаемых (см. раздел «Поддерживаемые типы ассетов»). Это не ошибка — так плагин сообщает, что для этого конкретного типа ассета логика извлечения ещё не реализована.

**Кнопка "Export to file…" показывает "Nothing to export for this asset."**
Значит, для этого конкретного ассета ни один из разделов (Structure/Graph/Design) не дал содержательного результата — например, ассет пуст, только что создан и ещё не содержит логики, либо для его типа некоторые вкладки в принципе неприменимы. Проверьте содержимое вкладок прямо в окне плагина перед экспортом: если там пусто или стоит placeholder-текст вроде "No Data found" — экспортировать действительно нечего.

**После нажатия "Export to file…" ничего не произошло.**
Если вы закрыли системный диалог выбора файла без сохранения (например, нажали «Отмена») — это штатное поведение, никакой ошибки нет, плагин просто молча ничего не делает. Если же вы точно выбрали файл и нажали «Сохранить», но уведомления об успехе не появилось — проверьте, есть ли у вас права на запись в выбранную папку (например, попробуйте сохранить в папку `Saved` вашего проекта, которая предлагается по умолчанию), и посмотрите Output Log редактора — туда плагин пишет предупреждение, если запись файла не удалась.

**Вложенные виджеты в выводе обрываются раньше, чем ожидалось / не показывают глубоко вложенные элементы.**
Это, скорее всего, следствие настройки **Widget Recursion Depth** (Edit → Project Settings → Plugins → Blueprint Reader). Увеличьте значение глубины либо снимите галочку **Restrict Widget Recursion Depth**, чтобы полностью снять ограничение — подробности в разделе «Настройки плагина».

**Значения свойств вложенного пользовательского виджета (`WBP_*`) выглядят как значения по умолчанию, а не то, что реально стоит на экране.**
Плагин показывает для вложенных пользовательских виджетов раздел "Exposed Properties (set on this instance)" — там перечислены свойства, значения которых были явно переопределены именно в этом конкретном месте использования (в этом экземпляре), в отличие от значения по умолчанию для всего класса виджета. Если свойство не меняли на данном конкретном экземпляре — оно там просто не отображается, потому что не несёт дополнительной информации сверх значения по умолчанию.

**Плагин не собирается / редактор ругается на модуль BlueprintReader после обновления версии движка.**
Плагин поддерживает Unreal Engine 5.7 и 5.8 "из коробки" на едином исходном коде — специальных действий с вашей стороны не требуется. Если вы используете более раннюю версию движка (5.6 и старше), сборка может завершиться ошибкой — в этом случае стоит либо обновить движок до поддерживаемой версии, либо проверить страницу проекта на GitHub на предмет более старой версии плагина, совместимой именно с вашей версией движка.

**Я нашёл баг или хочу предложить поддержку нового типа ассета — куда обращаться?**
Поскольку BlueprintReader — open-source проект, лучший способ — открыть issue в репозитории проекта на GitHub. Ссылку на репозиторий можно найти как во всплывающем окне для неподдерживаемых ассетов (кнопка "Check for updates"), так и на странице продукта на Fab.
