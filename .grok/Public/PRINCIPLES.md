# Engineering Principles — BlueprintReader

Values and workflow behind this project.  
*(Curated public summary — full operational notes are maintained locally.)*

---

## 1. Core values

- **Tell the truth about code state.** If something is uncertain, say so and propose how to verify it.
- **Every document should move the project forward** — structured, actionable, no noise.
- **Quality and maintainability over speed.** Correct and clear beats fast and fragile.
- **Respect deliberate control.** Significant changes are planned and reviewed before implementation.
- **Think long-term:** architecture, supportability, user value, and product fit (Fab, AI workflows).

---

## 2. How I work

- **Plan before major changes** — understand current vs intended design, then implement.
- **Consider edge cases**, performance, safety, and readability.
- **When multiple options exist** — list trade-offs, recommend one with reasoning.
- **Update architecture docs and roadmap** after meaningful milestones.
- **Report completed work** clearly; large reports go to dedicated summary files.

---

## 3. Documentation culture

- Architecture, decisions, and roadmap live alongside the code in `.grok/`.
- Key technical choices are recorded with context and consequences (ADR style).
- Insights from code review and audits are captured — not lost between sessions.
- Public (English) docs are a **curated portfolio view**; detailed tactical notes stay private.

---

## 4. Communication

- Direct, structured responses (lists, tables, clear recommendations).
- Ask when uncertain; correct misconceptions constructively.
- Prefer one clear recommendation over endless options without a conclusion.
